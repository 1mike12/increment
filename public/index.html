Running

Design Strategy

I decided to use node since it was a language that I was familiar with, and which also had fairly good I/O performance.

I rely on a query builder called knex to make database setup and easier and save time from having to write raw sql.

I basically wanted to make a daemon that ran on an interval that was less than 10 seconds, while traffic from post requests were buffered until the proper time to persist was reached. Since the throughput of sqlite writes is much higher than that of node itself, I felt it was ok to arbitrarily run the daemon every 5 seconds. Initially, through testing, the server could only handle ~400 requests/ second on my machine. I then changed the main app to take advantage of the multiple cpus available. To do this, I created child processes and make sure that they communicated all traffic to the master process. And as before, when the interval was reached, the master process would persist its data to the database. After converting the server to utilize clusters, I was able to reach 2200 req / second
